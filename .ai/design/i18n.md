# Internationalization (i18n)

> Status: active | Max lines: 100

## Overview

Chinese-first bilingual app. Default locale: `zh`. Supported: `zh`, `en`.

## Implementation

Use Flutter's official `flutter_localizations` + `intl` package with ARB files.

### Setup in `MaterialApp`
```dart
MaterialApp(
  locale: const Locale('zh'),
  supportedLocales: const [Locale('zh'), Locale('en')],
  localizationsDelegates: [
    AppLocalizations.delegate,
    GlobalMaterialLocalizations.delegate,
    GlobalWidgetsLocalizations.delegate,
    GlobalCupertinoLocalizations.delegate,
  ],
)
```

### ARB File Structure
```
lib/l10n/
├── app_zh.arb    ← source of truth (Chinese, default)
└── app_en.arb    ← English translation
```

Generate with: `flutter gen-l10n`

## Rules for Developers

1. **NEVER** hardcode user-facing strings in Dart code
2. All new strings: add to `app_zh.arb` first, then `app_en.arb`
3. Use `AppLocalizations.of(context)!.keyName` or short alias `context.l10n.keyName`
4. ARB key naming: `camelCase`, descriptive (e.g., `serverListTitle`, `deleteConfirmMessage`)
5. Plurals and placeholders: use ICU message format in ARB
6. Keep translations concise — UI space is limited on mobile

## ARB Key Categories

| Prefix | Category | Example |
|--------|----------|---------|
| `common` | Shared (OK, Cancel, Error) | `commonCancel` |
| `server` | Server management | `serverAddTitle` |
| `terminal` | SSH terminal | `terminalReconnecting` |
| `file` | File manager | `fileDeleteConfirm` |
| `docker` | Docker | `dockerContainerStopped` |
| `script` | Scripts & snippets | `scriptRunSuccess` |
| `settings` | Settings pages | `settingsSecurityTitle` |
| `sync` | WebDAV sync | `syncConflictMessage` |
| `update` | App update | `updateAvailable` |

## Appearance and Language Switching

- Settings → General → Language
- Options: "跟随系统" (follow system) / "中文" / "English"
- Default: follow system, fallback to zh if system locale unsupported
- Change takes effect immediately (no restart)
- Appearance page localizes dynamic color and popular theme color option labels
- Dynamic color is selected inside the same picker as manual seed colors
- Terminal appearance settings localize font, size, foreground, background, and dashboard labels
- Home page localizes the more menu tooltip and placeholder layout action

## l10n Configuration

File: `l10n.yaml`
```yaml
arb-dir: lib/l10n
template-arb-file: app_zh.arb
output-localization-file: app_localizations.dart
output-dir: lib/l10n
```

Generated files go to `lib/l10n/` (NOT synthetic package). Import via `package:orbita/l10n/app_localizations.dart`.

## Implementation Status

- Theme mode persistence: `shared_preferences` key `theme_mode`
- Dynamic color persistence: `shared_preferences` key `dynamic_color`
- Theme seed persistence: `shared_preferences` key `theme_seed`
- Terminal appearance persistence: `shared_preferences` keys prefixed with `terminal_`
- Locale persistence: `shared_preferences` key `locale`
- Providers: `themeModeProvider`, `dynamicColorProvider`, `themeSeedProvider`, `terminalAppearanceProvider`, `localeProvider`
- Appearance page: `/settings/appearance` — theme mode, unified color picker, terminal appearance, language
